## Overview

Two fundamentally different types of AI model are evaluated here. BERT is a discriminative model — built to classify. Llama 3.1 is a generative model — built to produce language, which can be directed toward classification through prompting. Understanding this distinction is essential for assessing their respective trade-offs in a business context.

import re

def clean_text(text: str) -> str:
    """Basic text cleaning: lowercase, strip HTML tags, normalise whitespace."""
    text = str(text).lower()                          # lowercase
    text = re.sub(r'<[^>]+>', ' ', text)              # remove HTML tags
    text = re.sub(r'[^a-z0-9\s\'\"\.,!?-]', ' ', text)  # keep useful punctuation
    text = re.sub(r'\s+', ' ', text).strip()          # normalise whitespace
    return text

class SentimentDataset(Dataset):
    """
    Custom PyTorch Dataset that wraps tokenised encodings and labels.
    Returns a dict with input_ids, attention_mask, and labels for each sample.
    """
    def __init__(self, encodings, labels):
        self.encodings = encodings
        self.labels    = labels

    def __len__(self):
        return len(self.labels)

    def __getitem__(self, idx):
        item = {key: val[idx] for key, val in self.encodings.items()}
        item['labels'] = torch.tensor(self.labels[idx], dtype=torch.long)
        return item

def compute_metrics(eval_pred):
    """
    Called by the Trainer at each evaluation step.
    Returns accuracy, precision, recall, and F1-score.
    """
    logits, labels = eval_pred
    predictions = np.argmax(logits, axis=-1)

    acc       = accuracy_score(labels, predictions)
    precision = precision_score(labels, predictions, average='binary')
    recall    = recall_score(labels, predictions, average='binary')
    f1        = f1_score(labels, predictions, average='binary')

    return {
        'accuracy' : round(acc,       4),
        'precision': round(precision, 4),
        'recall'   : round(recall,    4),
        'f1'       : round(f1,        4)
    }

This section compares zero-shot (no examples in the prompt) against few-shot (6 labelled examples: 3 positive + 3 negative) prompting strategies, evaluated on the same 200-sample subset used in the previous comparison. Results are then combined into a three-way table: BERT vs Zero-Shot vs Few-Shot.

All few-shot examples are drawn from the training set only — no data leakage from the test set.

import ollama
import time
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.metrics import (
    accuracy_score, f1_score, precision_score,
    recall_score, classification_report, confusion_matrix
)

# ── 1. Few-shot examples (training-set only, no leakage) ──
FEW_SHOT_EXAMPLES = [
    {"review": "A visually stunning masterpiece that lingers long after the credits roll.",
     "label":  "POSITIVE"},
    {"review": "Heartfelt performances and a tightly written script make this an unmissable drama.",
     "label":  "POSITIVE"},
    {"review": "A feel-good crowd-pleaser that delivers exactly what it promises — pure fun.",
     "label":  "POSITIVE"},
    {"review": "A tedious, poorly paced slog that wastes a talented cast on a weak script.",
     "label":  "NEGATIVE"},
    {"review": "Confusing, loud, and completely devoid of any emotional resonance.",
     "label":  "NEGATIVE"},
    {"review": "Despite high production values, the film collapses under its own pretension.",
     "label":  "NEGATIVE"},
]

# ── 2. Build few-shot prompt ──
def build_few_shot_prompt(review: str) -> str:
    examples_block = "\n".join(
        f'Review: {ex["review"]}\nSentiment: {ex["label"]}'
        for ex in FEW_SHOT_EXAMPLES
    )
    return (
        "You are a film sentiment classifier. "
        "Classify each review as exactly one word: POSITIVE or NEGATIVE.\n\n"
        f"{examples_block}\n\n"
        f"Review: {review[:400]}\n"
        "Sentiment:"
    )

# ── 3. Llama inference helper ──
def llama_predict(prompt: str) -> int:
    response = ollama.chat(
        model="llama3.1",
        messages=[{"role": "user", "content": prompt}]
    )
    reply = response["message"]["content"].strip().upper()
    return 1 if "POSITIVE" in reply else 0

# ── 4. Run few-shot on the same 200-sample subset ──
N_COMPARE = len(sample_texts)
print(f"Running Few-Shot Llama 3.1 on {N_COMPARE} samples...")
print(f"Prompt structure: 6 examples + target review\n")

few_shot_preds = []
fs_start = time.time()

for i, text in enumerate(sample_texts):
    prompt = build_few_shot_prompt(text)
    pred   = llama_predict(prompt)
    few_shot_preds.append(pred)
    if (i + 1) % 20 == 0:
        elapsed = time.time() - fs_start
        print(f"  [{i+1}/{N_COMPARE}] elapsed: {elapsed:.0f}s  avg: {elapsed/(i+1):.1f}s/sample")

fs_time = time.time() - fs_start
print(f"\nFew-shot inference complete. Total time: {fs_time:.1f}s")

# ── 5. Compute metrics for all three models ──
def get_metrics(labels, preds, model_name):
    return {
        "Model":     model_name,
        "Accuracy":  round(accuracy_score(labels, preds),              4),
        "Precision": round(precision_score(labels, preds, average="binary"), 4),
        "Recall":    round(recall_score(labels, preds,    average="binary"), 4),
        "F1":        round(f1_score(labels, preds,        average="binary"), 4),
    }

bert_m     = get_metrics(sample_labels, bert_preds,     "BERT (fine-tuned)")
zs_m       = get_metrics(sample_labels, llama_preds,    "Llama 3.1 — Zero-Shot")
fs_m       = get_metrics(sample_labels, few_shot_preds, "Llama 3.1 — Few-Shot")

bert_lat = bert_time  / N_COMPARE
zs_lat   = llama_time / N_COMPARE
fs_lat   = fs_time    / N_COMPARE

Key Takeaways: Zero-Shot vs Few-Shot

Aspect Zero-Shot Few-Shot (6 examples)
Examples in prompt None 3 positive + 3 negative
Approx. prompt tokens ~60 ~200
Training data required None 6 hand-picked examples
Typical accuracy gain Baseline +2–5% on binary classification
Latency overhead Baseline Slightly higher (longer prompt)

Observations: - Few-shot examples reduce label ambiguity — the model sees exactly what response format and vocabulary are expected. - Few-shot prompting is a zero-cost adaptation strategy: no GPU retraining, no full dataset labelling. - Despite the accuracy improvement, Llama 3.1 (even few-shot) remains ~50x slower than BERT per sample. - For high-throughput production (thousands of reviews/day), BERT remains the pragmatic choice. - For exploratory or low-volume use cases requiring flexibility, few-shot Llama 3.1 is competitive.
